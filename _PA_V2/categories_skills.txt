# PRODUCERS AVENUE - CATEGORIES & SKILLS

## Overview

Users can select multiple categories that describe their roles in the music industry. Each category expands to show relevant skills and specializations. This system enables precise matching for collaborations, accurate search results, and effective marketplace filtering.

## Category Selection System

### How It Works
1. During registration or profile editing, users see all main categories
2. Users can check **multiple categories** (no limit)
3. When a category is checked, it expands to show relevant skills
4. Users can select **multiple skills** within each category
5. "Other" option with text input available (requires admin approval)
6. Genre preferences and experience level selected separately

### Database Structure
- `user_categories` (many-to-many): Links users to main categories
- `user_skills` (many-to-many): Links users to specific skills
- `user_genres` (many-to-many): Links users to genre preferences
- `user_profile`: Stores experience level and other metadata

## Main Categories

Users can select as many categories as apply to them.

## Experience Level

**Purpose:** Help set expectations and find appropriate collaborators/clients

**Selection:** Single choice (one level selected)

### Experience Levels

**Beginner**
- **Duration:** 0-2 years
- **Description:** Learning the craft, building skills, early projects
- **Typical Activities:** Practice, tutorials, first releases, experimentation
- **Looking For:** Learning opportunities, mentorship, beginner-friendly collabs

**Intermediate**
- **Duration:** 2-5 years
- **Description:** Consistent work, building portfolio, developing style
- **Typical Activities:** Regular releases, client work, collaborations, skill refinement
- **Looking For:** Growth opportunities, fair-paid work, networking

**Professional**
- **Duration:** 5-10 years
- **Description:** Established career, quality portfolio, reliable income
- **Typical Activities:** Regular clients, professional releases, industry connections
- **Looking For:** Quality projects, professional rates, industry recognition

**Veteran**
- **Duration:** 10+ years
- **Description:** Industry recognition, extensive experience, established reputation
- **Typical Activities:** High-profile projects, mentoring, industry influence
- **Looking For:** Premium projects, partnerships, legacy building

**Note:** Experience level is self-reported and serves as a general indicator. Skills and portfolio matter more than years.

## Profile Completeness

To maximize discoverability and collaboration opportunities, users should:

### Required (Cannot use platform without):
- Username
- Email (verified)
- At least one main category

### Highly Recommended:
- At least 3 skills selected
- At least 3 genres selected
- Experience level
- Bio/description (150+ characters)
- Profile picture
- Location (city/country)

### Optional but Helpful:
- Portfolio links (website, SoundCloud, Spotify, etc.)
- Social media links
- Equipment/software list
- Availability status
- Collaboration preferences
- Rate/pricing information

### Completeness Score:
Users see a "Profile Completeness" percentage:
- 0-30%: "Get Started" (red)
- 31-60%: "Basic Profile" (yellow)
- 61-85%: "Strong Profile" (blue)
- 86-100%: "Complete Profile" (green)

Benefits of complete profiles:
- Higher search ranking
- More collaboration matches
- Better client trust
- Featured opportunities
- Professional appearance

## Matching & Search

### How Categories/Skills Affect Matching

**Collab Matcher:**
- Matches based on complementary skills
- Example: Producers matched with vocalists, guitarists with drummers
- Genre overlap increases match score
- Experience level considered (prevent mismatches)

**Search:**
- Users can filter by any category, skill, or genre
- Multi-select filters (AND/OR logic)
- Search by keyword matches categories and skills
- Auto-suggest skills as user types

**Recommendations:**
- Platform suggests collaborators based on:
  - Skills needed for user's work
  - Genre overlap
  - Location proximity (optional)
  - Experience level compatibility
  - Activity level

## Admin Approval for "Other" Fields

When a user enters a custom skill/genre/category:

1. Submission stored as "pending approval"
2. Admin notification sent
3. Admin reviews:
   - Is it legitimate?
   - Does it fit existing categories?
   - Is it duplicative?
4. Admin approves or denies (with reason)
5. If approved:
   - Added to user's profile
   - Optionally added to master list for all users
6. User notified of decision

This prevents spam while allowing legitimate new skills/genres to be added.
