===========================================
PAYMENT FLOW
===========================================

STRIPE FLOW:
1. User adds items to cart (CartContext)
2. User goes to /cart, clicks checkout
3. Frontend calls /api/stripe/checkout
4. Stripe session created with metadata
5. User redirected to Stripe checkout
6. After payment, Stripe webhook triggers /api/stripe/webhook
7. Webhook creates orders, transactions, notifications, download records
8. User redirected to /checkout/success

PAYPAL FLOW:
1. User adds items to cart (CartContext)
2. User goes to /cart, clicks checkout
3. Frontend calls /api/paypal/checkout
4. PayPal order created with custom_id
5. User redirected to PayPal
6. After approval, frontend calls /api/paypal/capture
7. Capture creates orders, transactions, notifications, download records
8. User redirected to /checkout/success
9. PayPal webhook confirms payment asynchronously

DOWNLOAD FLOW:
1. User purchases product
2. product_downloads record created (5 downloads, 30-day expiry)
3. User clicks download from /orders page
4. GET /api/products/download/[productId]
5. System verifies purchase, checks limits, generates signed URL
6. Downloads remaining decremented
7. Download logged in download_logs table
8. Signed URL expires after 1 hour
