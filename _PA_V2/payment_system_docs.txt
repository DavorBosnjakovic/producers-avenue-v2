## PAYMENT ARCHITECTURE OVERVIEW

Producers Avenue uses a dual payment system:
- **Stripe Connect (Destination Charges)** - For card payments
- **PayPal Payouts** - For PayPal payments

Both systems use escrow periods to protect buyers and sellers, with fully automated payouts after release.

   - Identity (government ID)

**Payment Flow:**

**Key Features:**
- âœ… Stripe handles all KYC (Know Your Customer) verification

- âœ… Stripe Connect account = verified identity + bank account
- âœ… Late chargebacks: Stripe reverses transfer from seller's account

## 3. ESCROW RULES

### Products (Digital Downloads)

**Escrow Period: 7 DAYS (Fixed)**

```
Purchase Date â†’ Day 1
Download link sent automatically
â†“
7 days pass (buyer can report issues)
â†“
Day 8: Money auto-released to seller's available balance
â†“
Seller can request payout immediately
```

**Why 7 Days:**
- Buyer has time to download and test files
- Buyer can report if files are corrupted/broken
- Buyer can file DMCA if copyrighted content
- Catches 95%+ of disputes before payout

**Auto-Release:**
- No action needed from buyer
- No action needed from seller
- Automatic after 7 days
- Funds move from "Pending" to "Available"

---

### Services (Freelance Work)

**Escrow Period: UNTIL BUYER CONFIRMS DELIVERY (Max 7 Days)**

```
Service Purchased (Order Created)
â†“
Seller marks as "Delivered" + uploads files
â†“
Buyer receives notification
â†“

  â†’ Money released IMMEDIATELY to seller
â†“

  â†’ After 7 days, money auto-released
â†“
OPTION 3: Buyer clicks "Request Revision"
  â†’ Escrow continues, seller must revise
  â†’ Process repeats
```

**Revision Handling:**
```
1. Seller delivers work
2. Buyer requests revision (if within included revisions)
3. Escrow continues
4. Seller submits revised work
5. Buyer accepts OR 7 days pass â†’ Release
```

**Maximum Escrow Time:**
- **7 days from delivery** (not from purchase)
- If buyer doesn't respond in 7 days = Auto-accept
- Prevents buyers from holding money indefinitely

**Example Timeline:**
```
Day 1: Service purchased
Day 5: Seller delivers work
Day 5: Escrow starts counting (7 days from delivery)

```

## 10. DISPUTE RESOLUTION

### Buyer-Seller Disputes

**Process:**
1. Buyer contacts seller directly (via messages)
2. Seller has 48 hours to respond
3. If unresolved, buyer can open formal dispute
4. Platform reviews:
   - Messages between parties
   - Product description vs actual files
   - Preview vs delivered content
5. Decision made within 3 business days
6. If refund approved: Deducted from seller's balance

**Dispute Categories:**
- Files corrupted/don't work
- Files significantly different from preview
- Copyright infringement (DMCA)
- Technical download issues

---

### DMCA Claims

**Process:**
1. Copyright holder files DMCA notice
2. Product immediately taken down
3. Seller notified
4. Seller can file counter-notice
5. If no counter-notice: Refund issued, seller charged
6. If counter-notice: Legal process follows

**Automatic Actions:**
- Product delisted immediately
- All pending transactions refunded
- Seller's pending balance frozen
- Future sales prevented until resolved

```sql
  paypal_verified BOOLEAN DEFAULT false,
  
  -- Calculated totals (in cents)
  balance_total INTEGER DEFAULT 0,
  balance_available INTEGER DEFAULT 0,

  -- Lifetime stats (in cents)
  lifetime_earnings_stripe INTEGER DEFAULT 0,
  lifetime_earnings_paypal INTEGER DEFAULT 0,

  lifetime_withdrawn_paypal INTEGER DEFAULT 0,
```

### Transactions Table

```sql
CREATE TABLE transactions (
  id UUID PRIMARY KEY,
  
  -- Parties
  buyer_id UUID REFERENCES auth.users(id),
  seller_id UUID REFERENCES auth.users(id),
  
  -- Transaction type
  transaction_type TEXT, -- 'product_purchase', 'service_purchase', etc.

  -- Amounts (in cents)
  amount_total INTEGER NOT NULL, -- What buyer paid

  -- Commission snapshot
  commission_rate DECIMAL(5,2), -- e.g., 0.30 for 30%
  seller_tier TEXT, -- Subscription tier at time of sale
  
  -- Payment processor
  payment_processor TEXT, -- 'stripe' or 'paypal'
  payment_method TEXT, -- 'card', 'apple_pay', 'paypal', etc.
  
  -- Processor IDs
  stripe_payment_intent_id TEXT,
  stripe_transfer_id TEXT, -- ID of transfer to seller's Stripe Connect
  paypal_transaction_id TEXT,
  paypal_order_id TEXT,
  
  -- Status
  status TEXT, -- 'pending', 'completed', 'refunded', 'disputed'
  completed_at TIMESTAMPTZ,
  
  -- Escrow
  escrow_release_date DATE, -- When funds released (7 days)
  escrow_released BOOLEAN DEFAULT false,

  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

### Daily Escrow Release Cron Job

```javascript
// Runs every day at midnight
async function releaseEscrowedFunds() {
  const today = new Date();
  
  // Products: 7 days from purchase
  const productsToRelease = await db.transactions.findMany({
    where: {
      escrow_released: false,
      escrow_release_date: { lte: today },
      transaction_type: 'product_purchase'
    }
  });
  
  // Services: 7 days from delivery OR buyer accepted
  const servicesToRelease = await db.orders.findMany({
    where: {
      escrow_released: false,
      OR: [
        { buyer_accepted: true },
        { auto_accept_date: { lte: today } }
      ]
    }
  });
  
  // Release all
  for (const transaction of productsToRelease) {
    await releaseToAvailable(transaction);
  }
  
  for (const order of servicesToRelease) {
    await releaseToAvailable(order.transaction_id);
  }
}
```

## 16. EDGE CASES & SOLUTIONS

### Seller Account Closed Before Payout

**Scenario:** Seller closes account with pending balance

**Solution:**
```
1. Account closure prevented if balance > $0
2. Must withdraw all funds first
3. OR wait for admin to manually refund buyers
```

---

### Buyer Downloads Then Disputes

**Scenario:** Buyer downloads product, then files chargeback

**Solution:**
```
1. During escrow: Money still held, refund processed
2. After escrow: Chargeback handled by Stripe/PayPal
   - Stripe: Reversed from seller
   - PayPal: Deducted from wallet or debt created
3. Log download timestamp as evidence
```

---

### Seller Never Delivers Service

**Scenario:** Seller takes money, never delivers

**Solution:**
```
1. Buyer waits delivery_days
2. Buyer opens dispute: "Seller not delivering"
3. Platform investigates
4. If seller hasn't responded: Auto-refund
5. Money still in escrow, easy refund
6. Seller account suspended
```

---

### Mass Chargeback Attack

**Scenario:** Bad actor creates fake accounts, files many chargebacks

**Solution:**
```
1. Stripe/PayPal detect fraud patterns
2. Chargebacks reversed if fraudulent
3. Platform monitor chargeback rates per seller
4. Threshold: >2% = investigation
5. Repeated offenders: Account termination
```
